# JH Toolbox 고도화 기획안

## 1. 현재 서비스 스냅샷

- 제품 성격: 브라우저 안에서 끝나는 browser-only 툴박스
- 구현 축: PDF, 이미지, 비디오/오디오, OCR, 데이터, 웹/캡처
- 현재 규모: 8개 카테고리, 총 117개 툴 ID
- 카테고리별 툴 수
  - PDF: 25
  - 이미지: 28
  - 비디오: 21
  - 오디오: 12
  - 파일/데이터: 10
  - 스크린 캡처: 6
  - 웹: 5
  - OCR: 2
- 구조 특징
  - 공통 UI 허브: `components/tool-ui/tool-workbench.tsx`
  - 처리 엔진 분리: `lib/processors/pdf.ts`, `image.ts`, `media.ts`, `ocr.ts`, `data.ts`, `web.ts`
  - 이미 에디터형 경험이 있는 툴: PDF 페이지 편집, 오디오 웨이브폼 컷, 브라우저 캡처, 이미지 크롭
  - 레거시/호환용 숨김 툴 ID도 존재: 12개

## 2. 현재 강점

- 업로드를 서버로 보내지 않는 로컬 처리 포지셔닝이 분명하다.
- PDF, 이미지, 비디오, 오디오, OCR까지 한 앱 안에서 다루는 범위가 넓다.
- `ffmpeg.wasm`, `pdf-lib`, `tesseract.js` 등으로 기능 신뢰도가 기본 이상이다.
- 이미 일부 툴은 “옵션 입력”이 아니라 “직접 편집” 경험으로 올라와 있다.
- 정적 배포 구조라 배포와 확장이 비교적 단순하다.

## 3. 현재 한계

### UX 관점

- 많은 툴이 여전히 “파일 업로드 → 숫자 옵션 입력 → 실행” 패턴에 머물러 있다.
- 툴 간 인터랙션 깊이가 들쭉날쭉하다.
  - 예: 오디오는 웨이브폼 에디터가 있는데, 비디오/워터마크/리사이즈는 상대적으로 폼 중심이다.
- 결과 화면은 다운로드 중심이고, 재편집/비교/재사용 흐름은 약하다.
- 초보자 입장에서 “무슨 툴을 골라야 하는지”보다 “카테고리 안에서 뭘 해야 하는지” 안내가 부족하다.

### 제품 관점

- 툴 수는 많지만, 자주 쓰는 시나리오를 하나의 워크플로우로 묶는 기능이 적다.
- 프리셋 저장, 최근 작업 복원, 템플릿 재사용 같은 재방문 장치가 약하다.
- 모바일/저사양 환경에서 무거운 WASM 초기 로딩을 체감할 가능성이 크다.

### 운영 관점

- 테스트가 `ui-shell.spec.ts` 중심이라 카테고리별 핵심 플로우 회귀 방지가 더 필요하다.
- 툴 성공률, 처리 시간, 중도 이탈 지점 같은 제품 지표 관찰 기반이 없다.

## 4. 제품 목표

### 북극성

`브라우저 안에서 가장 빠르게 결과를 얻는 올인원 작업 툴박스`

### 핵심 목표

1. 단순 변환 앱에서 “편집 가능한 작업 공간”으로 진화
2. 1회성 방문자를 재방문 사용자로 전환
3. 툴 수 증가보다 대표 시나리오 완성도를 우선
4. 무거운 처리도 “느리지만 믿을 수 있는” 경험으로 정리

## 5. 우선 추진 전략

### 전략 A. 고빈도 툴의 에디터화

현재 잘 만든 에디터형 경험을 다른 카테고리로 확장한다.

- 이미지
  - 크롭처럼 워터마크, 텍스트 추가, 경계선 추가도 캔버스 기반 미리보기로 전환
  - 배경 제거/블러는 전후 비교 슬라이더 추가
- 비디오
  - 비디오 크롭을 프레임 오버레이 에디터로 전환
  - 컷/썸네일/워터마크를 타임라인 또는 스토리보드 기반으로 고도화
- PDF
  - redact, sign, edit-pdf를 드래그형 오버레이 편집기로 통일

### 전략 B. 워크플로우 묶음 상품화

단일 툴이 아니라 “문제 해결 묶음”으로 재구성한다.

- 문서 패키지
  - PDF 병합 → 페이지 정리 → 압축 → 워터마크
- 쇼츠/콘텐츠 패키지
  - 비디오 리사이즈 → 크롭 → 무음/오디오 분리 → GIF/WEBP 출력
- 이미지 패키지
  - 배경 제거 → 리사이즈 → 포맷 변환 → 워터마크
- 데이터 패키지
  - CSV/JSON/XML 상호 변환 + 스키마 검증 + 샘플 미리보기

### 전략 C. 재사용성 강화

- 사용자 프리셋 저장
- 최근 작업 복원
- “다음에도 이 설정으로 실행” 템플릿
- 카테고리별 추천 프리셋 세트
  - SNS 비율
  - 전자문서 표준 크기
  - 유튜브/쇼츠/릴스 출력 규격

### 전략 D. 신뢰도와 처리 체감 개선

- 무거운 처리 시작 전 예상 시간/메모리 경고
- 취소, 재시도, 단계별 진행률, 실패 원인 메시지 개선
- 결과물 검수용 before/after compare 추가
- 모바일과 저사양 기기용 경량 모드

### 전략 E. 탐색 구조 고도화

- 카테고리 탐색보다 “하려는 일” 기준 진입 강화
- 예:
  - PDF 합치기
  - 사진 배경 제거
  - 영상 비율 맞추기
  - 음성만 추출하기
- 홈에서 최근 사용, 인기 툴, 새 툴, 추천 워크플로우를 분리 노출

## 6. 카테고리별 고도화 제안

### PDF

- P1
  - PDF 분할/삭제/회전도 썸네일 기반 선택 UI 제공
  - redact/edit/sign 툴 오버레이 편집 통합
  - PDF 비교를 side-by-side diff + 요약 리포트로 개선
- P2
  - OCR 적용 후 searchable PDF 생성
  - 문서 일괄 처리 모드
  - 서식 템플릿형 워터마크/서명 프리셋

### 이미지

- P1
  - 워터마크/텍스트/보더를 캔버스 에디터로 전환
  - 전/후 비교, 확대 보기, 히스토리 1-step undo
  - SNS/마켓플레이스 목적별 출력 프리셋
- P2
  - 배경 제거 품질 모드 추가
  - 이미지 컬러 팔레트 추출 결과를 디자인 토큰 형식으로 내보내기
  - 배치 리사이즈/배치 포맷 변환 UX 개선

### 비디오

- P1
  - 비디오 크롭/리사이즈/워터마크 실시간 프레임 미리보기
  - 썸네일 생성에 프레임 스크러버 추가
  - trim 도구 분리 또는 video-convert 안에 간단 컷 편집 포함
- P2
  - subtitle burn-in/extract
  - 배치 큐 처리
  - output preset을 shorts / reel / youtube / web-banner 식으로 추상화

### 오디오

- P1
  - audio-merge에 다중 클립 순서 편집 UI 추가
  - loudness normalize / silence trim / fade preset
  - 오디오 변환 결과 비교 청취
- P2
  - 노이즈 감소
  - 간단한 EQ/voice boost 프리셋

### OCR

- P1
  - OCR 결과를 문단/줄/블록 단위로 하이라이트
  - 이미지 일부 영역만 OCR
- P2
  - 표 추출을 CSV/XLSX로 바로 연결
  - 언어팩 추천 및 다국어 혼합 인식 UX

### 파일/데이터

- P1
  - 변환 전 샘플 preview
  - 열 매핑 UI
  - JSON/XML pretty/minify/validate 도구 추가
- P2
  - CSV schema validation
  - 대용량 분할/병합 preset

### 웹/스크린

- P1
  - URL 캡처 결과에 주석/크롭/하이라이트 추가
  - 스크린샷 이후 바로 blur/redact/crop 워크플로우 연결
  - capture 실패 시 CORS/권한 원인 안내
- P2
  - 모바일/태블릿/데스크톱 디바이스 프리셋 캡처
  - 브라우저 캡처 후 PDF/이미지 동시 출력

## 7. 우선순위 로드맵

### Phase 1. 0~4주

- 목표: 체감 개선이 큰 고빈도 툴 정리
- 범위
  - image-crop 패턴을 image watermark/text/border로 확장
  - video-crop 미리보기 에디터 도입
  - PDF redact/sign/edit-pdf 오버레이 통합 설계
  - 결과 before/after compare 공통 컴포넌트 추가
- 성공 기준
  - 대표 에디터형 툴 3개 이상 추가
  - 관련 Playwright 시나리오 확장

### Phase 2. 4~8주

- 목표: 단일 툴 모음에서 워크플로우 제품으로 전환
- 범위
  - 프리셋 저장
  - 최근 작업 복원
  - 워크플로우 번들 진입 추가
  - 홈/카테고리 탐색 고도화
- 성공 기준
  - 재방문율과 preset 사용률 측정 가능

### Phase 3. 8~12주

- 목표: 신뢰도와 프로 사용자 가치 확대
- 범위
  - batch queue
  - advanced preview
  - searchable PDF, subtitle, data validation 등 고급 기능
  - 성능/메모리 가이드와 취소/복구 체계
- 성공 기준
  - 대용량 작업 실패율 감소
  - 완료까지 걸리는 평균 시간 단축

## 8. 공통 플랫폼 과제

### UX 플랫폼

- 공통 editor shell
  - 좌측 미리보기
  - 우측 속성 패널
  - 하단 결과 비교
- 공통 preset 저장 구조

### 처리 플랫폼

- 작업 큐와 취소 토큰

- 큰 파일 처리 시 메모리 가드레일
- 실패 로그 수집 포인트 정의

### 품질 플랫폼

- 카테고리별 golden path E2E 확장
- processor 단위 smoke test

  - 첫 처리 시작 시간
  - 50MB/200MB 파일 기준 완료 시간

## 9. KPI 제안

- 활성화
  - 방문 후 첫 실행까지 시간
  - 첫 다운로드 전환율
- 사용성
  - 툴별 완료율
  - 실패율
  - 재실행 비율
- 재방문
  - 최근 사용 툴 재진입률
  - preset 저장/재사용률
- 성능
  - WASM 초기 로딩 시간
  - 대용량 파일 중단율

## 10. 가장 먼저 실행할 추천 과제

바로 체감이 크고 현재 코드 구조와도 잘 맞는 5가지는 아래 순서가 좋다.

1. 이미지 편집군 공통 캔버스 에디터화
2. 비디오 크롭/워터마크 프리뷰 에디터
3. PDF redact/sign/edit-pdf 오버레이 통합
4. preset 저장 + 최근 작업 복원
5. before/after compare와 실패 메시지 고도화

## 11. 결론

JH Toolbox는 이미 “툴 수”는 충분하다. 다음 단계는 더 많은 기능 추가보다, 자주 쓰는 툴을 에디터형 경험과 워크플로우형 경험으로 끌어올리는 것이다.  
즉, `많은 기능`에서 `완성도 높은 작업 경험`으로 초점을 옮기는 것이 가장 큰 성장 포인트다.
